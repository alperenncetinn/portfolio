Entity Framework ve ORM: Veritabanı İşlemlerinde Modern Yaklaşım

.NET Entity Framework ORM Database SQL Server

Entity Framework, .NET ekosisteminde veritabanı işlemlerini kolaylaştıran güçlü bir ORM (Object-Relational Mapping) kütüphanesidir. Bu yazıda Entity Framework'ün temellerini, ORM kavramını ve modern veritabanı işlemlerindeki yerini detaylı olarak inceleyeceğiz.

Entity Framework ORM Mimarisi ve Veritabanı Entegrasyonu

Entity Framework Nedir?

Entity Framework, Microsoft tarafından geliştirilen bir ORM (Object-Relational Mapper) kütüphanesidir. .NET geliştiricilerinin nesnelerini kullanarak veritabanlarıyla etkileşim kurmasını sağlar ve geleneksel SQL kodlamasının büyük bir kısmını ortadan kaldırır.

ORM Nedir?

ORM (Object-Relational Mapping), nesne yönelimli programlama dilleri ile ilişkisel veritabanları arasında bir köprü görevi gören teknolojidir. ORM araçları:

  • Veritabanı tablolarını programlama dili sınıflarına eşler
  • SQL sorguları yazmadan veritabanı işlemleri yapmanızı sağlar
  • Kod üzerinden veritabanı şemasını yönetir
  • Farklı veritabanı sistemleri arasında taşınabilirlik sunar

Entity Framework'ün Desteklediği Veritabanları

SQL Server

Microsoft'un ana veritabanı sistemi. En iyi performans ve uyumluluk.

UseSqlServer(connectionString)

MySQL

Açık kaynak veritabanı sistemi. Web uygulamaları için popüler.

UseMySQL(connectionString)

PostgreSQL

Gelişmiş açık kaynak veritabanı. JSON desteği güçlü.

UseNpgsql(connectionString)

SQLite

Hafif dosya tabanlı veritabanı. Test ve prototip için ideal.

UseSqlite(connectionString)

Oracle

Kurumsal uygulamalar için güçlü veritabanı sistemi.

UseOracle(connectionString)

In-Memory

Test senaryoları için bellek tabanlı veritabanı.

UseInMemoryDatabase("TestDb")

Entity Framework Yaklaşımları

Code First

Kod yazarak başlayın, veritabanı otomatik olarak oluşturulur.

Avantajları:

  • Kod odaklı geliştirme
  • Version control ile uyumlu
  • Migration desteği

Dezavantajları:

  • Karmaşık veritabanları için zor
  • Performans optimizasyonu sınırlı

Database First

Mevcut veritabanından başlayın, kod otomatik olarak üretilir.

Avantajları:

  • Mevcut veritabanları için ideal
  • DBA kontrolü korunur
  • Karmaşık şemalar desteklenir

Dezavantajları:

  • Kod üzerinden şema değişikliği zor
  • Veritabanına bağımlılık

Model First

Visual designer ile model oluşturun, kod ve veritabanı üretilir.

Avantajları:

  • Görsel tasarım desteği
  • Hızlı prototipleme
  • Takım çalışması için uygun

Dezavantajları:

  • Visual Studio bağımlılığı
  • Karmaşık projeler için sınırlı

Temel Kavramlar

Entity Framework'ün Temel Bileşenleri: Entity, DbContext, DbSet ve Navigation Properties

Entity

Veritabanı tablosunu temsil eden C# sınıfı. Her entity bir tablo satırına karşılık gelir.

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
    public int CategoryId { get; set; }
    public Category Category { get; set; }
}

DbContext

Veritabanı bağlantısını ve entity'leri yöneten ana sınıf. Session ve Unit of Work pattern'lerini uygular.

public class AppDbContext : DbContext
{
    public DbSet<Product> Products { get; set; }
    public DbSet<Category> Categories { get; set; }
    
    protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
    {
        optionsBuilder.UseSqlServer(connectionString);
    }
}

DbSet

Belirli bir entity türü için veritabanı tablosuna erişim sağlayan koleksiyon.

using (var context = new AppDbContext())
{
    // Tüm ürünleri getir
    var products = context.Products.ToList();
    
    // Yeni ürün ekle
    context.Products.Add(new Product { Name = "Laptop" });
    context.SaveChanges();
}

Navigation Properties

Entity'ler arasındaki ilişkileri temsil eden özellikler. Foreign key ilişkilerini kodda yansıtır.

public class Category
{
    public int Id { get; set; }
    public string Name { get; set; }
    
    // One-to-Many ilişki
    public ICollection<Product> Products { get; set; }
}

Entity Framework'ün Avantajları

Hızlı Geliştirme

SQL yazma ihtiyacını ortadan kaldırarak geliştirme sürecini hızlandırır.

Güvenlik

SQL Injection saldırılarına karşı otomatik koruma sağlar.

Taşınabilirlik

Farklı veritabanı sistemleri arasında kolayca geçiş yapabilirsiniz.

Migration Desteği

Veritabanı şema değişikliklerini kod üzerinden yönetir.

Lazy Loading

İhtiyaç duyulduğunda verileri yükleyerek performansı optimize eder.

Change Tracking

Entity'lerdeki değişiklikleri otomatik olarak takip eder.

Dikkat Edilmesi Gerekenler

Performans Optimizasyonu

  • N+1 Problem: Eager loading veya projection kullanın
  • Gereksiz Sorgular: AsNoTracking() metodunu kullanın
  • Büyük Veri Setleri: Pagination ve filtering uygulayın
  • Connection Pooling: DbContext yaşam döngüsünü doğru yönetin

En İyi Uygulamalar

Repository Pattern Kullanın

DbContext'i doğrudan kullanmak yerine repository pattern ile soyutlayın.

Dependency Injection

DbContext'i DI container ile yönetin, manuel instansiation yapmayın.

Migration Stratejisi

Production'da migration'ları manuel olarak kontrol edin.

Configuration

Fluent API ile entity konfigürasyonlarını ayrı sınıflarda yapın.

Sonuç

Entity Framework, modern .NET uygulamalarında veritabanı işlemlerini kolaylaştıran güçlü bir araçtır. ORM yaklaşımı sayesinde geliştiriciler SQL yazma karmaşıklığından kurtularak iş mantığına odaklanabilirler.

Doğru kullanıldığında Entity Framework, hem geliştirme sürecini hızlandırır hem de kod kalitesini artırır. Ancak performans optimizasyonu ve best practice'lere dikkat edilmesi önemlidir.
